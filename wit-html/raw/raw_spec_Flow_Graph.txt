With the layout established, the details that follow customise the objects.

<pre>NEXTOBJECT:COLOUR=rgb(255,230,205);BCOLOUR=rgb(215,155,0);CORNER_RADIUS=15;TIP=</pre>
<h1>WIT Shortcuts</h1>
Raw data listing the shortcuts.  Each shortcut is associated with a menu item.

<pre>NEXTOBJECT:COLOUR=rgb(255,230,205);BCOLOUR=rgb(215,155,0);CORNER_RADIUS=15;TIP=</pre>
<h1>WIT Tooltips</h1>
The text of the data for each tooltip explaining what a menu item does.

<pre>NEXTOBJECT:COLOUR=rgb(255,230,205);BCOLOUR=rgb(215,155,0);CORNER_RADIUS=15;TIP=</pre>
<h1>WIT Menus</h1>
Raw data that specifies the full tree-like structure of the Audacity menus.  There is an entry in the raw data for each menu item.

<pre>NEXTOBJECT:COLOUR=rgb(255,230,205);BCOLOUR=rgb(215,155,0);CORNER_RADIUS=15;TIP=</pre>
<h1>WIT</h1>
<p>Wit is a web application that shows "What is That?" for many of the user interface fetures in Audacity.  It also has a hidden feature where it can generate images and tables for use in the Audacity manual.</p>
<p>Wit consumes information stored in our Wiki as raw JSON data, and produces images and formatted tables which can be uploaded to the wiki used for our manual.



<pre>NEXTOBJECT:COLOUR=rgb(215,230,210);BCOLOUR=rgb(130,180,100);TIP=</pre>
<h1>Toolbar Images</h1>
Annotated images of 8 of our toolbars.  These are extracted from the master image and then labelled.
<pre>NEXTOBJECT:COLOUR=rgb(215,230,210);BCOLOUR=rgb(130,180,100);TIP=</pre>
<h1>Templates</h1>
The auto-generated templates that include, for example, image maps with detailed information about the boundary boxes of menu items.
<pre>NEXTOBJECT:COLOUR=rgb(215,230,210);BCOLOUR=rgb(130,180,100);TIP=</pre>
<h1>Big Pages</h1>
Autogenerated formatted wiki-text for the large pages that, for example, list and describe the Audacity scripting API.  That table lists every command supported by Audacity Scripting.  Another big page is the page of effects.
<pre>NEXTOBJECT:COLOUR=rgb(215,230,210);BCOLOUR=rgb(130,180,100);TIP=</pre>
<h1>Menu Images</h1>
Around 50 images of menus in Audacity, showing different submenus open.

<pre>NEXTOBJECT:COLOUR=rgb(220,230,250);BCOLOUR=rgb(108,140,190);TIP=</pre>
<h1>Audacity</h1>
Audacity can generate raw data used by WIT.  For example, it inspects its own data about the menu structure and creates a file for that.  It also generates the 'list of boxes' for the locations and dimesnions of toolbars.
<pre>NEXTOBJECT:COLOUR=rgb(220,230,250);BCOLOUR=rgb(108,140,190);TIP=</pre>
<h1>Scripting</h1>
<pre>NEXTOBJECT:COLOUR=rgb(220,230,250);BCOLOUR=rgb(108,140,190);TIP=</pre>
<h1>Preferences Images</h1>
Audacity Screenshot Tool can capture images of all the preferences screens.
<pre>NEXTOBJECT:COLOUR=rgb(220,230,250);BCOLOUR=rgb(108,140,190);TIP=</pre>
<h1>EGA Images</h1>
Audacity Screenshot Tool can capture images of all the Effects, Generators and Analysers (EGAs).

<pre>NEXTOBJECT:COLOUR=rgb(220,230,250);BCOLOUR=rgb(108,140,190);TIP=</pre>
<h1>Track Images</h1>
Track Images can be generated by Audacity scripting.  The script sets up different configurations of tracks and then calls the screenshot tool to capture an image of each.


<pre>NEXTOBJECT:COLOUR=rgb(225,210,230);BCOLOUR=rgb(150,115,165);TIP=</pre>
<h1>File Upload</h1>
Small numbers of auto-generated image files are uploaded by hand.  For larger numbers, pywikibot is used.

<pre>NEXTOBJECT:COLOUR=rgb(225,210,230);BCOLOUR=rgb(150,115,165);TIP=</pre>
<h1>Cut &amp; Paste</h1>
Various tabular data is always pasted into the manual by hand.
<pre>NEXTOBJECT:COLOUR=rgb(225,210,230);BCOLOUR=rgb(150,115,165);CORNER_RADIUS=15;TIP=</pre>
<h1>Alpha Manual</h1>
The manual for the current alpha edition of Audacity is maintained in wiki.  When it is ready the mw2html script is used to convert it to static html pages, which can be included with Audacity itself, and also uploaded to the current online copy.
